## 1. 적용 원칙

### 1.1 기능 구성 원칙

- **기능정의서 우선:** 화면 구성은 `feature-spec.md`의 1Depth → 2Depth → 기능분류 계층을 기준으로 결정한다.
- **이미지는 스타일 참고용:** 참고 이미지(`docs/ref_images/`)는 개별 기능의 시각 스타일 참고용이다. 이미지에 보이는 탭이나 다른 메뉴가 같은 화면에 존재해야 한다는 의미가 아니다.
- **기능 묶음 우선:** 동일 2Depth에 속하는 기능은 하나의 화면에 묶어 제공한다. 기능분류(기능ID 기준)가 탭 또는 섹션의 단위가 된다.

### 1.2 UX 구성 원칙

- **업무 흐름 순 배치:** 탭 순서와 기능 배치는 실제 운영자의 업무 순서를 따른다.
- **반복 유틸은 우측 날개로:** 여러 화면에서 반복 호출되는 기능(PII 검출 등)은 우측 유틸 날개 패널로 전역 제공한다.
- **메뉴 구조 고정:** 확정된 메뉴 구조(IA)는 화면 작업 중 임의로 변경하지 않는다. 변경이 필요하면 사전 협의 후 반영한다.

### 1.3 구현 원칙

- HTML / CSS / Vanilla JS 전용. 백엔드 없음, 더미 데이터 사용.
- 모든 클릭 요소, 탭, 모달, 토글, 드롭다운은 JS로 동작하게 구현한다.
- 인터랙션 없는 버튼을 남기지 않는다. 준비 중이더라도 Toast, 모달, 상태 변경 등의 반응을 제공한다.
- 공통 레이아웃(헤더, 사이드바, Workspace Tabs)은 `layout.js`가 렌더링한다.

## 2. 메뉴 구조 확정 (IA)

> **출처:** 기능정의서 엑셀 `IA_관리자` 시트 재파싱 (병합 셀 해제 후 확인)  
> **규칙:** 사이드바 = 1Depth > 2Depth / 콘텐츠 탭 = 기능분류 / 화면 내 UI = 기능상세

```
■ 데이터 관리                          (1Depth)
  ├── PII 패턴 정보 관리               (2Depth)
  │     탭: 내장 PII 패턴 | 커스텀 패턴
  ├── 익명화 처리                       (2Depth)
  │     (탭 없음 — 단일 2단 분할 화면)
  ├── 일괄 처리                         (2Depth)
  │     (탭 없음 — 스텝 플로우 화면)
  └── 데이터 전처리 관리               (2Depth)
        탭: 전처리 단어사전 | 제외 단어사전 | NER 분리 접미사 |
            NER 지역명 | KMS 분류 | 유틸

■ 벡터 DB 관리                         (1Depth)
  ├── FAQ 생성 관리                     (2Depth)
  │     탭: FAQ 생성 | 검토·승인 | 컬렉션 조회
  └── 문서 정보 관리                   (2Depth)
        (탭 없음 — 목록 + 슬라이드 상세)

■ 데이터 거버넌스                      (1Depth)
  └── FAQ 거버넌스 관리                (2Depth)
        탭(업무흐름 순): PDF 문서 | PDF 파싱 | PDF 검수 |
                        정규식 전처리 | LLM 전처리 |
                        PDF 임베딩/컬렉션 | RAG 시스템 컬렉션

■ 채팅 관리                            (1Depth)
  ├── 채팅 세션 관리                   (2Depth)
  │     (탭 없음 — Master-Detail 레이아웃)
  └── 카카오톡 채널연동 서버           (2Depth)
        (탭 없음 — 단일 상태/설정 카드)
```

### 사이드바 동작 규칙

- **1Depth 클릭:** 서브메뉴 토글 + 첫 번째 2Depth 자동 선택
- **2Depth 클릭:** 해당 화면을 Workspace Tab에 추가 및 활성화
- **사이드바 접기:** 240px ↔ 64px. 접힘 시 아이콘만 표시, hover 시 툴팁 표시
- **Workspace Tabs:** 브라우저 탭과 유사한 열린 화면 탭 (sessionStorage 유지, 닫기/전체닫기 가능)

## 3. 공통 UX 패턴

### 3.1 탭 + CRUD (가장 많이 사용)

```
[ 구조 ]
상단 탭 메뉴
─────────────────────────────
탭 콘텐츠 영역
  ├── 툴바: 검색 입력 + [+ 추가] 버튼
  ├── 목록 테이블
  │     컬럼: 활성 토글 | 항목명 | 값/설명 | 수정 | 삭제
  └── 폼: 인라인 펼침 or 모달 팝업

**적용 화면:** PII 패턴 정보 관리, 데이터 전처리 관리

### 3.2 슬라이드 상세 패널 (Master-Detail)

```
[ 구조 ]
┌───────────────────────┬──────────────────┐
│  목록 영역 (60%)      │  상세 패널 (40%) │
│  ─────────────────    │  ─────────────   │
│  필터 + 검색          │  선택 항목 정보  │
│  테이블 (클릭 가능)   │  편집 폼        │
│                       │  액션 버튼      │
└───────────────────────┴──────────────────┘
```

**동작:** 행 클릭 → 우측 상세 패널 슬라이드인. 바깥 클릭 또는 닫기 버튼으로 닫힘.

### 3.3 스텝 플로우 (단계형 워크플로우)

  (활성)   (비활성)   (비활성)   (비활성)

현재 스텝 콘텐츠 영역

**적용 화면:** 일괄 처리 (파일업로드 → 옵션설정 → 처리실행 → 결과확인)

```
[ 구조 ]
탭 순서 = 업무 처리 순서
Tab1 → Tab2 → Tab3 → ... → TabN

**특징:** 탭 헤더에 처리 상태 뱃지(완료/대기/오류) 표시

### 3.6 공통 Confirm / Toast / Modal

| 패턴 | 용도 | 구현 |
|------|------|------|
| Confirm 팝업 | 삭제·초기화 등 비가역 작업 | `AdminConfirm.show()` |

### 3.7 진행 상태 패턴 (비동기 Task)

```
[실행 버튼 클릭]
    ↓
버튼 내 스피너 + 비활성화
    ↓
Task 생성 → 진행률 바 표시 (실시간 폴링 시뮬레이션)
    ↓
완료 → Toast 알림 + 결과 화면 전환
오류 → 오류 메시지 Toast + 재시도 버튼
```

**적용 화면:** 일괄 처리, FAQ 생성, PDF 임베딩

## 4. 공통 우측 유틸 날개 패널

### 4.1 개념

여러 화면에서 반복 호출되는 기능을 우측 고정 아이콘 바에 배치한다.  
로그아웃 같은 시스템 유틸이 아닌, **업무 지원 도구** 성격의 기능을 제공한다.

### 4.2 레이아웃 위치

```
┌──────┬──────────────────────────────┬────┐
│ Side │                              │ 🔍 │
│  bar │       메인 콘텐츠 영역       │ 📋 │ ← 우측 날개 (48px 고정)
│      │                              │ 📚 │
│      │                              │ 💾 │
└──────┴──────────────────────────────┴────┘
                                       ↑
                          아이콘 클릭 시 슬라이드 패널 (320px)
```

- 헤더 아래, 화면 우측에 48px 너비로 고정
- 메인 콘텐츠 영역을 밀지 않음 (overlay 방식)
- 모든 화면에 공통 노출

### 4.3 유틸 항목

| 순서 | 아이콘 | 기능명 | 내용 | 관련 기능ID |
|------|--------|--------|------|------------|
| 1 | `bi-shield-check` | PII 빠른 검출 | 텍스트 입력 후 즉시 PII 탐지. 설정 없이 빠른 단건 확인용 | FU-09-004 |
| 2 | `bi-list-ul` | PII 범례 | 검출 유형 목록·설명 참조 카드 | FU-09-003 |
| 3 | `bi-book` | 단어사전 조회 | 전처리·제외 단어 빠른 검색·참조 | FU-08-001~009 |
| 4 | `bi-database-check` | 컬렉션 상태 | RAG 컬렉션 현황 요약 (문서 수, 활성 상태) | FU-06-001 |

### 4.4 동작 규칙

- 아이콘 클릭 → 해당 패널 슬라이드인 (오른쪽에서 진입)
- 이미 열린 패널의 아이콘 재클릭 → 패널 닫힘
- 다른 아이콘 클릭 → 현재 패널 닫고 새 패널 오픈
- 배경(overlay) 클릭 또는 패널 내 닫기(×) 버튼 → 닫힘
- 활성 아이콘: Primary Blue 배경 강조

### 4.5 익명화 처리 화면과의 역할 구분

| 구분 | 우측 날개 PII 검출 | 익명화 처리 화면 |
|------|------------------|----------------|
| 목적 | 빠른 단건 확인 (설정 없음) | 상세 패턴 설정 포함 완전한 테스트 |
| 패턴 선택 | 없음 (기본 설정 사용) | 내장/커스텀 패턴 직접 선택 |
| 결과 | 간단 하이라이팅 | 상세 검출 목록 + 범례 포함 |

### Screen 03 — 데이터 관리 > PII 패턴 정보 관리

- **커스텀 패턴 탭:**
  - [+ 추가] 버튼 → 인라인 폼 (패턴이름 / 정규식 / 문맥단어)
  - 토글 + 패턴명 + 정규식 + 수정 아이콘 + 삭제 아이콘
  - 삭제 → Confirm 팝업
  - 수정 → 모달 팝업
- **상태:** ✅ 완료

### Screen 05 — 데이터 관리 > 일괄 처리

- **파일:** `batch-processing.html`
- **레이아웃:** 스텝 플로우 + 하단 결과 그리드
- **스텝 구성:**

```
STEP 1. 파일 업로드
  └── 파일 선택 버튼 + 파일명 표시 + [업로드] 버튼

STEP 2. 처리 설정 (설정 펼치기/접기 패널)
  ├── 대상 컬럼 선택 (체크박스)
  ├── 강제 마스킹 컬럼 선택
  ├── 처리 옵션 (Ollama LLM 사용, 배치 크기)
  ├── 자동 제외 활성화 → PII 밀도 임계값·범위 설정
  ├── 내장 패턴 목록 (체크박스 리스트)
  ├── 커스텀 패턴 목록 (체크박스 리스트)
  └── 제외 카테고리 (추가/불러오기/저장 버튼)

STEP 3. 일괄 처리 시작
  └── [일괄 처리 시작] 버튼 → 실시간 진행률 바

STEP 4. 결과 확인
  ├── 통계 카드 (전체/완료/PII검출/미검출/제외 건수)
  ├── 데이터 그리드 (행 클릭 → 우측 슬라이드 상세 패널)
  │     상세 패널: 원본 데이터 / 마스킹 결과 / 감지 PII 목록
  ├── [CSV로 저장] (마스킹/미검출/제외 각각)
  └── 처리 통계 요약
```

- **기능 ID:** FU-09-006 ~ FU-09-020
